# Tacit SaaS Product Strategy

## 1. Positioning

**Tacit is an AI content operating system for experts, consultants, founders, and technical teams who already have deep knowledge but no time to turn it into consistent LinkedIn content.**

Tacit's category:

**Knowledge-to-content platform for LinkedIn.**

Tacit is not just another AI LinkedIn writer. The core thesis is that serious professionals already have valuable knowledge hidden in documents, presentations, reports, notes, frameworks, client work, training material, and internal expertise. Tacit turns that material into publishable LinkedIn insight, while keeping the user in control of review, approval, scheduling, and brand voice.

## 2. Core Promise

Tacit helps knowledge-rich professionals publish consistently on LinkedIn without starting from a blank page.

The product takes existing expertise as input and produces:

- Original LinkedIn post ideas.
- Insight-led post drafts.
- Multiple post variants.
- Campaign sequences.
- Scheduled content calendars.
- Performance feedback.

The user still owns the final judgment. Tacit does the research, extraction, drafting, organization, and scheduling workflow.

## 3. Differentiation

Most LinkedIn growth tools start from generic prompts, viral templates, or trend inspiration.

Tacit starts from the user's own knowledge base.

| Dimension | Generic LinkedIn AI tools | Tacit |
|---|---|---|
| Main input | Prompt, topic, or viral post inspiration | User documents, notes, frameworks, and expert material |
| Core value | Generate more posts faster | Turn existing expertise into authority-building content |
| Output style | Often generic or trend-driven | Source-grounded, voice-aware, expertise-led |
| Best fit | Creators chasing consistency and reach | Experts, consultants, founders, and technical teams |
| Risk | Sounds like everyone else | Preserves domain knowledge and point of view |

Tacit can include familiar execution features such as scheduling, queues, analytics, and calendar planning, but its defensible center is the knowledge-to-content engine.

## 4. Target Users

### Primary ICP

Solo consultants, independent experts, founders, senior specialists, trainers, coaches, and B2B operators who:

- Have strong expertise but inconsistent LinkedIn presence.
- Own valuable documents, frameworks, reports, or training materials.
- Need thought leadership to support trust, authority, and inbound opportunities.
- Do not want to sound like a generic AI content account.

### Secondary ICP

Small B2B teams and technical companies that want their experts to publish more consistently, including:

- Engineering consultancies.
- Training companies.
- SaaS founders.
- Professional services firms.
- Niche technical agencies.

### Not Ideal for v1

- High-volume social media agencies.
- Cold outreach teams.
- Lead scraping users.
- Creators who only want viral trend replication.
- Consumer influencers focused on entertainment content.

## 5. Product Pillars

### 1. Knowledge Ingestion

Users upload or paste source material:

- PDFs.
- Word documents.
- Slide decks.
- Notes.
- Articles.
- Reports.
- Training material.
- SOPs.
- Case studies.
- Topic briefs.

Tacit stores sources, extracts text, and keeps the original source linked to any generated insight.

### 2. Insight Extraction

Tacit identifies content-worthy material inside the source:

- Practical lessons.
- Mistakes and warnings.
- Contrarian takes.
- Frameworks.
- Checklists.
- Stories.
- Technical explanations.
- Executive summaries.
- Common misconceptions.
- Actionable recommendations.

The key is not summarization. The key is finding ideas worth publishing.

### 3. LinkedIn Draft Generation

Tacit generates post drafts from extracted insights.

Supported post types for v1:

- Educational post.
- Story-led post.
- Contrarian post.
- Practical checklist.
- Framework post.
- Opinion post.
- Soft promotional post.

Each draft should include:

- Hook.
- Body.
- Clear idea progression.
- Optional CTA.
- Source reference.
- Channel or audience fit.

### 4. Voice and Brand Profile

Users define how they want to sound.

Inputs:

- Past LinkedIn posts.
- Brand tone notes.
- Target audience.
- Banned phrases.
- Preferred topics.
- Writing examples.

Output:

- A structured voice profile.
- Reusable generation rules.
- Channel-specific writing style.

### 5. Review Queue

All AI output goes into a review queue.

Users can:

- Edit drafts.
- Compare variants.
- Approve.
- Reject.
- Regenerate.
- Assign a campaign.
- Schedule.

Tacit should never hide the review step. The product should feel like a publishing desk, not an autopilot.

### 6. Calendar and Scheduling

Users plan consistent LinkedIn activity.

Features:

- Visual calendar.
- Scheduled posts.
- Draft backlog.
- Campaign view.
- Content type labels.
- Channel labels.

Publishing should use official LinkedIn APIs only.

### 7. Analytics and Feedback

Tacit helps users learn what kind of expertise performs.

Track:

- Impressions.
- Likes.
- Comments.
- Shares.
- Engagement rate.
- Topic.
- Post type.
- Source document.
- Campaign.

The key analytics question:

**Which knowledge assets and themes are creating the most audience response?**

## 6. MVP Scope

The MVP should prove the core loop before building a full SaaS surface.

### MVP User Journey

1. User creates an account.
2. User creates a brand/voice profile.
3. User uploads a PDF/DOCX document or pastes source text.
4. Tacit screens and extracts insights.
5. User selects an insight.
6. Tacit generates three LinkedIn post variants.
7. User edits and approves one draft.
8. User schedules the post inside Tacit or copies it for manual publishing.
9. User sees the draft in a content queue/calendar.

### MVP Features

- Email/password authentication.
- Workspace/account model.
- PDF/DOCX source upload and text extraction.
- Source text paste mode.
- Insight extraction.
- Post generation.
- Voice profile creation.
- Draft review queue.
- Basic calendar and internal scheduling.
- Manual export/copy post.
- Admin-safe environment configuration.

### MVP Features to Defer

- Full LinkedIn OAuth publishing.
- Advanced analytics.
- Lead database.
- Auto-DM workflows.
- Auto-commenting.
- Chrome extension.
- Carousel generation.
- Team approvals.
- Agency client management.
- Complex billing plans.

## 7. v1 SaaS Features

After the MVP proves the knowledge-to-content loop, v1 can add:

- LinkedIn OAuth.
- Direct scheduling and publishing.
- Post analytics.
- Campaign builder.
- Product/service promotion sequences.
- Multi-workspace support.
- Subscription billing.
- Basic team seats.
- Content library search.
- Saved prompt/style templates.

## 8. Product Principles

- **Source-grounded first:** Every good draft should be traceable to a source, topic, or campaign brief.
- **Expertise over virality:** Reach matters, but the product should optimize for authority and credibility.
- **Human approval always:** AI drafts should not be published without review.
- **Official APIs only:** No scraping, cookie-based auth, browser automation, or risky LinkedIn behavior.
- **Calm professional UX:** Tacit should feel like a serious workbench for experts, not a flashy growth hack.
- **Voice matters:** Generic AI language is a product defect.
- **Small teams first:** Build for individuals and small B2B teams before agencies.

## 9. Suggested Pricing Hypothesis

Pricing should reflect the value of saving expert time and producing authority-building content.

### Starter

For solo professionals.

- Limited monthly source uploads.
- Limited AI drafts.
- One LinkedIn profile.
- Manual export.
- Basic calendar.

Potential price: **$19-$39/month**.

### Pro

For consultants, founders, and active creators.

- More source uploads.
- More AI drafts.
- Voice profile.
- Scheduling.
- Analytics.
- Campaigns.

Potential price: **$49-$99/month**.

### Team

For small B2B companies.

- Multiple users.
- Multiple brand profiles.
- Approval workflows.
- Shared content library.
- Company page support.

Potential price: **$149-$299/month**.

## 11. First Development Step

Start with a prototype that proves the core content engine:

**PDF/DOCX upload or paste -> insight extraction -> three LinkedIn draft variants -> editable review queue -> internal scheduling.**

Recommended first technical milestone:

- Create the application skeleton.
- Add database schema for users, workspaces, sources, insights, voice profiles, and posts.
- Add PDF/DOCX upload and source text paste flow.
- Add OpenAI service interface.
- Add prompt templates.
- Build a simple dashboard showing extracted insights, draft variants, and scheduled posts.

This milestone should work even before LinkedIn OAuth, billing, and analytics are added.

## 12. Resolved Product Decisions

- Final name: **Tacit**.
- First ICP: **solo consultants and independent founders**.
- First input format: **PDF/DOCX upload from day one, plus paste text**.
- First output: **schedule inside Tacit, with copy/export available as a fallback**.
- AI provider: **OpenAI**.
- Stack: **FastAPI + Jinja2 + HTMX + Tailwind + PostgreSQL**.
- Billing: **after private beta**.

## 13. Remaining Open Decisions

- Should the first private beta include only individual LinkedIn profiles, or also company pages?
- Should scheduled posts publish automatically later, or should Tacit keep a final manual confirmation step?
- Which OpenAI model should be the default for generation once implementation starts?
