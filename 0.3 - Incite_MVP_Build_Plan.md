# Incite MVP Build Plan

## 1. MVP Goal

Build the smallest usable SaaS prototype that proves Incite's core promise:

**A user can upload or paste expert material, extract publishable insights, generate LinkedIn post drafts in a defined voice, and manage those drafts in a review queue.**

The MVP is not trying to compete feature-for-feature with Taplio yet. It is proving Incite's unique wedge: knowledge-to-content for LinkedIn.

## 2. MVP Success Criteria

The MVP is successful if a test user can:

- Create an account and workspace.
- Complete the guided **Onboarding Flow** (Set voice profile &rarr; Upload or paste first source &rarr; Auto-generate first posts).
- Connect their LinkedIn profile or company page via OAuth.
- Add sources by either uploading files (PDF/DOCX/TXT/MD) or **pasting plain text**.
- Review extracted insights and generate LinkedIn post drafts.
- **Regenerate a single post** directly from the Calendar or Drafts screen with a different angle.
- View and manage post scheduling in a calendar dashboard.
- Sync and track **post performance metrics** (likes, comments, impressions) directly inside the History page.
- Manage workspace configuration (rename workspace, select timezone, connect/disconnect LinkedIn) via a **Settings page**.
- **Invite team members** to the workspace via email invitations.
- Browse, search, and filter past posts in a unified **Content Library**.

---

## 3. Recommended MVP Stack

### Application
**FastAPI + Jinja2 + HTML/JS/CSS**
- Python-first, keeping AI, document parsing, and database orchestration in one environment.
- Fast, clean, and interactive without the build overhead of a separate frontend framework.

### Database & ORM
**PostgreSQL + SQLAlchemy + Alembic**
- PostgreSQL for production/multi-tenant SaaS readiness.
- Alembic migrations to evolve schemas (adding metrics, settings, workspace team members, and invitations) cleanly.

### AI Provider
**OpenAI API**
- Accessing GPT-4o / GPT-4o-mini via a structured `AIClient` service interface to allow switching providers later.

## 4. Core Data Model

### `users`
Stores user identity.
- `id` (PK)
- `email` (Unique)
- `password_hash`
- `email_verified` (Boolean)
- `onboarding_complete` (Boolean)
- `google_id` (Nullable)
- `name`
- `picture_url`
- `created_at`

### `workspaces`
Represents an individual team or user's account space.
- `id` (PK)
- `name`
- `created_at`
- `timezone` (String, default "Asia/Dubai")

### `workspace_members`
Connects users to workspaces.
- `id` (PK)
- `workspace_id` (FK)
- `user_id` (FK)
- `role` ('owner' | 'admin' | 'member')
- `created_at`

### `workspace_invitations`
Tracks pending invitations to join a workspace.
- `id` (PK)
- `workspace_id` (FK)
- `email` (String)
- `token` (String, Unique)
- `status` ('pending' | 'accepted' | 'expired')
- `created_at`

### `voice_profiles`
Stores writing tones and brand alignment.
- `id` (PK)
- `workspace_id` (FK)
- `name`
- `target_audience`
- `tone_notes`
- `banned_phrases`
- `example_posts`
- `fingerprint_json`
- `created_at`
- `updated_at`

### `sources`
Stores uploaded files or pasted text.
- `id` (PK)
- `workspace_id` (FK)
- `title`
- `source_type` ('upload' | 'paste' | 'campaign')
- `raw_text`
- `file_path` (Nullable)
- `original_filename` (Nullable)
- `content_type` (Nullable)
- `generation_preferences_json`
- `status` ('ready' | 'failed')
- `created_at`

### `insights`
Ideas mined from the raw text.
- `id` (PK)
- `workspace_id` (FK)
- `source_id` (FK)
- `title`
- `insight_text`
- `angle`
- `audience`
- `confidence_score`
- `created_at`

### `posts`
LinkedIn drafts and published shares.
- `id` (PK)
- `workspace_id` (FK)
- `source_id` (FK, Nullable)
- `insight_id` (FK, Nullable)
- `voice_profile_id` (FK, Nullable)
- `variant_group_id`
- `variant_number`
- `draft_text`
- `status` ('draft' | 'approved' | 'scheduled' | 'posted' | 'publish_failed')
- `planned_publish_at` (Nullable)
- `scheduled_at` (Nullable)
- `posted_at` (Nullable)
- `linkedin_post_id` (Nullable)
- `publish_error` (Nullable)
- `likes_count` (Integer, default 0)
- `comments_count` (Integer, default 0)
- `impressions_count` (Integer, default 0)
- `last_metrics_sync` (Datetime, Nullable)
- `created_at`
- `updated_at`

## 5. MVP Screens & UX Flow

### 1. Login / Signup
- SaaS-style account creation and secure entry.
- Supports email/password or Google OAuth.

### 2. Onboarding Flow
- Strictly enforces guided step-by-step setup for new users:
  1. **Step 1 (Voice Profile)**: Input target audience, tone notes, and sample posts.
  2. **Step 2 (Add Knowledge)**: Paste text or upload their first PDF/DOCX/TXT/MD document.
  3. **Step 3 (Generate First Post)**: Automatically triggers AI processing of the source and outputs initial LinkedIn drafts, redirecting them straight to the Draft Queue.
- LANDING RULE: Any logged-in user with `onboarding_complete = False` is automatically redirected to `/onboarding`.

### 3. Dashboard
- Main operations center (only accessible after onboarding).
- Shows scheduled calendar preview, recent sources, draft queue counters, and social performance overview.

### 4. Sources Library
- View list of sources with extraction status cards.
- Add source modal with two tabs:
  - **Upload File**: Upload PDF, DOCX, TXT, or MD.
  - **Paste Text**: Simple plain text input area for notes, emails, or raw thoughts.

### 5. Draft Queue
- Grouped view of drafts (by variant groups).
- Actions: Edit inline, approve, reject, auto-schedule, or copy text.

### 6. Calendar & Scheduler
- Visual weekly/monthly layout of posts.
- Drag-and-reschedule interfaces or simple reschedule modals.
- **Regenerate Single Post**: A quick-action button on scheduled posts that rewrites the draft copy with a different angle or tweak without regenerating the entire variant group or source.

### 7. Post History & Analytics
- Lists all posted items.
- Displays post metrics: **likes**, **comments**, and **impressions**.
- Sync trigger: Fetches the latest stats from the LinkedIn API on page load (throttled to once every 10 minutes per post to prevent rate limits) or manual refresh.

### 8. Workspace Settings (NEW)
- Workspace profile (rename workspace, delete workspace).
- Timezone configuration (updates scheduling offsets).
- LinkedIn Channel Management: Connect, disconnect, and switch target destinations (personal profile or company pages).

### 9. Team Members (NEW)
- Lists active team members and their roles.
- Email invitation form that emails a secure registration token to colleagues to join the workspace.

### 10. Content Library (NEW)
- A searchable archive of all draft, scheduled, and published posts.
- Filters by source, status, target channel, and date range. Good for finding top-performing content to repurpose.

## 6. Prompt Templates Needed

### `insight_extraction.md`
- Mines raw document text for interesting angles, warnings, frameworks, and stories tailored to the voice target audience.

### `post_generation.md`
- Formulates 3 post variants matching formatting lengths, emoji guidelines, style fingerprint, and structure options.

### `post_regeneration.md` (NEW)
- Rewrites a single specific draft post based on user instructions (e.g., "Make it more punchy", "Change the hook", "Make it a warning post").

## 7. Development Phases

### Phase 1: Onboarding & Core Redirection
- Update root routes to check `user.onboarding_complete`.
- Finalize multi-step onboarding flow: voice setup &rarr; upload/paste source &rarr; trigger initial draft generation on completion &rarr; redirect to drafts.

### Phase 2: Input Enhancements & Plain Text
- Build "Paste Text" source creation input tab.
- Adapt document parsing services to process pasted text cleanly.

### Phase 3: Single Post Regeneration
- Build the `post_regeneration` service utilizing the AI client.
- Add a "Regenerate" UI button on the Calendar and Drafts screens to trigger a single-post rewrite overlay.

### Phase 4: Workspace Settings & Timezones
- Build the Workspace Settings page.
- Implement timezone settings modifying the scheduled execution times.
- Connect existing LinkedIn disconnect/reconnect flows to the new settings page.

### Phase 5: Team Invitations
- Build invitation schemas, token generation, and the mailer service.
- Implement the registration link landing page that matches tokens and associates new users with existing workspaces.

### Phase 6: Post Analytics & Feedback Loop
- Integrate the LinkedIn API metrics query (`socialActions` for likes/comments, and share analytics for impressions).
- Save stats in database and render metrics in the History page and Content Library.

### Phase 7: Unified Content Library
- Create the content library page with search parameters and database queries.
